Grep Command Overview

grep = Global Regular Expression Print

Used for: logs, config files, code, DevOps debugging

Syntax: grep [options] pattern [file(s)]

Meaning:

  • options → how grep behaves
  • pattern → what you are searching
  • file(s) → where to search

Common Options:

  • grep "pattern" file - Basic search
  • grep -i "pattern" file - Case-insensitive search
  • grep -c "pattern" file - Count matching lines
  • grep -l "pattern" file(s) - Print only matching file names
  • grep -w "word" file - Match whole words only
  • grep -o "pattern" file - Print only the match (not full line)
  • grep -n "pattern" file - Show line numbers
  • grep -v "pattern" file - Invert match (NOT matching)
  • grep "^pattern" file - Match start of line
  • grep "pattern$" file - Match end of line
  • grep -e "pattern1" -e "pattern2" file - Multiple patterns
  • grep -f pattern_file target_file - Read patterns from file
  • grep -A[n] pattern file - Print lines after match
  • grep -B[n] pattern file - Print lines before match
  • grep -C[n] pattern file - Print lines before + after
  • grep -r "pattern" directory - Search recursively
