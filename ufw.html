netstat

Network Statistics Command Reference

What is Netstat?

A powerful command-line utility for monitoring network connections, routing tables, interface statistics, and protocol information. Essential for system administrators and network troubleshooting.

macOS Users Note

Important: Flags need a command! Use netstat -a not just -a

Alternative: lsof -i -P is more modern on macOS

Common Flags

Flag Description Use Case
-a Show all connections (active + listening) Complete network overview
-t TCP connections only Web traffic, SSH
-u UDP connections only DNS, streaming, gaming
-n Numeric addresses (faster) Quick diagnostics
-p Show process ID and name Identify which app uses connection
-l Listening ports only Find open services
-r Display routing table Network path troubleshooting
-i Show network interfaces Interface statistics
-c Continuous output Real-time monitoring
-s Protocol statistics Network performance analysis

Essential Commands

View All Connections

netstat -an

Shows all connections with numeric addresses. Fastest way to see what's happening.

Beginner

Find Listening Services

netstat -ltn

Shows which TCP ports are listening. Great for checking if your server started.

Common

Check Specific Port

netstat -an | grep :8080

Find if anything is using port 8080. Replace with your port number.

Common

TCP with Programs

netstat -atp

See which programs are connected (Firefox, Chrome, etc.). May need sudo.

Advanced

Real-time Monitor

netstat -c

Continuous live feed of connections. Press Ctrl+C to stop.

Monitoring

macOS Alternative

lsof -i -P

Better for macOS. Shows programs using network with port numbers.

Mac Recommended

Connection States Explained

ESTABLISHED

Active connection with data flowing. This is a healthy, working connection.

LISTEN

Port is open and waiting for incoming connections. Your server is ready.

TIME_WAIT

Connection closed but waiting for late packets. Normal after closing.

CLOSE_WAIT

Remote side closed the connection, local side is about to close.

SYN_SENT

Your computer is trying to establish a connection to a server.

SYN_RECEIVED

Someone is trying to connect to your computer/server.

Common Use Cases

1. Check if Your Web Server Started

apple@Apples-MacBook-Pro ~ % netstat -an | grep :8080 tcp46 0 0 *.8080 *.* LISTEN # *.8080 in LISTEN state means your server is running!

2. Find Which App is Using a Port

apple@Apples-MacBook-Pro ~ % sudo lsof -i :3000 COMMAND PID USER FD TYPE DEVICE SIZE/OFF NODE NAME node 1234 apple 23u IPv6 0x1a2b3c4d5e6f 0t0 TCP *:3000 (LISTEN) # Node.js is using port 3000

3. Check SSH Connection

apple@Apples-MacBook-Pro ~ % netstat -an | grep :22 tcp46 0 0 *.22 *.* LISTEN # SSH server is listening on port 22

Quick Reference

Most Used Commands

  • netstat -an → All connections, numeric format
  • netstat -ltn → Listening TCP ports
  • netstat -r → Routing table
  • netstat -s → Protocol statistics
  • lsof -i -P → macOS: Show programs using network

Troubleshooting Tips

  • Always type netstat first, then add flags
  • On Mac, use -anL for listening ports (not -lt)
  • Add sudo if you get permission errors
  • Use | grep :PORT to search for specific ports
  • Combine flags: netstat -tulpn for maximum info

Understanding the Output

Column Meaning Example
Proto Protocol type tcp4, tcp6, udp4
Recv-Q Bytes in receive queue 0 (no backlog)
Send-Q Bytes in send queue 0 (sent successfully)
Local Address Your computer's IP:Port 192.168.0.174.55407
Foreign Address Remote server's IP:Port dns.google.443
State Connection status ESTABLISHED, LISTEN, etc.

Short Notes on Netstat Command

What is Netstat?

The netstat command in Linux is a robust tool for network troubleshooting and monitoring. It provides various options and outputs to display network connections, routing tables, interface statistics, masquerade connections, and multicast memberships. When executed, it displays detailed information about currently active TCP, UDP, and Unix socket connections.

Connection Types

  • TCP connections: Guarantee reliable data transfer and are typically used by applications such as web browsers and email clients
  • UDP connections: Used for faster, less reliable data transfer, often utilized in video streaming and gaming
  • Unix socket connections: Facilitate inter-process communication on the same machine, commonly seen in server applications

Socket States

Netstat not only shows sockets that are currently connected and exchanging data but also those that are listening and ready to accept incoming connections. These listening sockets are essential for server applications, as they await client connection requests.

Complete Options Reference

Option Description
--route, -r Displays the kernel routing tables. Useful for viewing the paths that network packets take. Similar to route -e output.
--groups, -g Shows multicast group membership information for both IPv4 and IPv6. Helpful for understanding multicast network configurations.
--interfaces, -i Displays a table of all network interfaces along with their statistics. Useful for monitoring the status and activity of each network interface.
--masquerade, -M Lists masqueraded connections, which are connections being modified by Network Address Translation (NAT).
--statistics, -s Provides summary statistics for each protocol. Useful for getting an overview of network activity and performance.
--verbose, -v Makes netstat provide more detailed output. Prints additional useful information about unconfigured address families.
--wide, -W Prevents truncation of IP addresses in the output. Useful to see the full address without cutoff, especially when scripting.
--numeric, -n Shows numerical addresses instead of resolving symbolic hostnames, port names, or usernames. Speeds up output by skipping DNS lookups.
--numeric-hosts Displays numerical host addresses only, while still resolving port names and usernames.
--numeric-ports Shows numerical port numbers only, while still resolving hostnames and usernames.
--numeric-users Displays numerical user IDs only, while still resolving hostnames and port names.
--protocol=family, -A Specifies the address families (protocols) for which connections are to be shown. Family can include: inet, unix, ipx, ax25, netrom, ddp.
--continuous, -c Continuously prints the selected information every second. Useful for real-time monitoring of network activity.
--extend, -e Displays additional information about each connection. Using twice (-ee) provides maximum detail.
--timers, -o Includes information related to networking timers, which can help diagnose timing-related network issues.
--program, -p Shows the PID and name of the program associated with each socket. Helps identify which processes are using network connections.
--listening, -l Displays only listening sockets, excluding active connections. Useful for checking which services are ready to accept incoming connections.
-F Prints routing information from the Forwarding Information Base (FIB). This is the default behavior for routing information.
-C Prints routing information from the route cache, providing insights into recent routing decisions made by the kernel.

Linux vs macOS Differences

Many Linux netstat options don't work on macOS:

  • -p (show PID/program) - Not available. Use lsof -i -P instead
  • -c (continuous) - Not supported on macOS
  • -o (timers) - Not supported on macOS
  • -x (Unix sockets) - Not supported on macOS
  • -C (route cache) - Not supported on macOS
  • -g (multicast groups) - Available on macOS
  • -r (routing table) - Available on macOS
  • -i (interfaces) - Available on macOS
  • -a (all connections) - Available on macOS

macOS Recommended Alternative:

apple@Apples-MacBook-Pro ~ % lsof -i -P COMMAND PID USER FD TYPE DEVICE SIZE/OFF NODE NAME Google 456 apple 45u IPv4 0x1a2b3c4d 0t0 TCP 192.168.0.174:55407->142.250.4.14:443 (ESTABLISHED) mysqld 1234 apple 3u IPv46 0x5e6f7a8b 0t0 TCP *:3306 (LISTEN) # Better for macOS - shows programs using network with ports

Key Takeaways

  • The netstat command is essential for network troubleshooting and monitoring
  • It displays TCP, UDP, and Unix socket connections with various details
  • Use -n flag for faster numeric output (no DNS resolution)
  • The -c flag provides continuous real-time monitoring
  • Combine -p with other flags to identify which programs are using connections
  • On macOS, consider using lsof -i as a modern alternative
  • Whether troubleshooting issues, monitoring performance, or managing configurations, netstat is indispensable

Pro Tips

  • Always use sudo with -p flag to see all process information
  • Combine flags for more specific output: netstat -tulpn shows TCP/UDP listening ports with programs
  • Use grep to filter results: netstat -an | grep :80
  • The -n flag speeds up output by avoiding DNS lookups
  • Check listening ports regularly as part of security audits
  • Use -s for protocol statistics to analyze network performance
